### [001] ✅ CUDA Incompatibility — RTX 5050 sm_120
**Timestamp:** Late April 2026 — Session 1
**Machine:** Lenovo (Build)
**Error:**
```
NVIDIA GeForce RTX 5050 Laptop GPU with CUDA capability sm_120 is not
compatible with the current PyTorch installation.
RuntimeError: CUDA error: no kernel image is available for execution on the device
```
**Root Cause:** RTX 5050 uses Blackwell architecture (sm_120). Stable PyTorch only compiled kernels up to sm_90. Nightly was installed but crashed on first actual kernel launch. `torch.cuda.is_available()` returned True but no kernel could actually execute.
**What was tried:**
- PyTorch nightly — detected GPU, crashed on encode
- ONNX Runtime GPU — eventually resolved it
**Resolution:** Implemented live CUDA smoke-test in `get_device()` that launches a tiny tensor operation before committing to GPU. Falls back silently to CPU on failure. ONNX Runtime GPU ultimately unlocked RTX 5050.

---

### [002] ✅ Character-Based Chunking — Context Destruction
**Timestamp:** Late April 2026 — Session 1
**Machine:** Lenovo (Build)
**Problem:** Original chunking sliced text every 700 characters with no awareness of sentence or paragraph boundaries. Chunks began and ended mid-sentence, destroying semantic context needed for lore queries.
**Bad output example:**
```
"...s of the Warmaster had already begun their descent"
```
**Resolution:** Replaced with NLTK sentence-aware chunking. Splits on paragraph breaks first, accumulates sentences to token budget (350 tokens), backs up by overlap_sentences (1-2) between chunks.

---

### [003] ✅ No File Type Dispatcher — Silent EPUB/AZW3/CBR Failures
**Timestamp:** Late April 2026 — Session 1
**Machine:** Lenovo (Build)
**Problem:** Original code used inline `if/else` on `.txt` vs everything else. EPUB, AZW3, CBR silently fell into the PDF branch and failed with no log entry.
**Resolution:** Clean `EXTRACTORS` dict mapping extension to function. Unsupported types now explicitly logged. Adding a new format requires one function and one dict entry.

---

### [004] ✅ No Failed File Tracking — Infinite Silent Retry Loop
**Timestamp:** Late April 2026 — Session 1
**Machine:** Lenovo (Build)
**Problem:** Files that failed extraction were silently skipped. Next run would attempt them again, fail again, with no record.
**Resolution:** Added `failed_files.json` and `failure_report.json`. Failed files excluded from future runs automatically. Retry is explicit and intentional.

### 1. The CUDA Compatibility Crisis (SM_120)
**Issue:** PyTorch stable lacked kernels for the RTX 5050.
**Solution:** Switched to **ONNX Runtime GPU**. Successfully resolved missing `cublasLt64_12.dll` issues by ensuring CUDA 12.x runtimes were in the path alongside CUDA 13.

**Issue:** Build crashed during WiFi drops.
**Solution:** Enforced strict offline mode using environment variables `TRANSFORMERS_OFFLINE="1"` and `HF_DATASETS_OFFLINE="1"`.

### 3. The "Denial of Wallet" Context Limit
**Issue:** Character-based chunking broke narrative context.
**Solution:** Implemented **Sentence-Aware Chunking** (NLTK punkt) with overlap.

### 4. The NumPy 2.0 Conflict
**Issue:** `AttributeError: _ARRAY_API not found`.
**Solution:** Downgraded to `numpy<2` (specifically `1.26.4`).

### 5. The System Path & Archive Ghost
**Issue:** WinRAR/rarfile dependencies were unreliable.
**Solution:** Migrated to a direct `subprocess` call to **7-Zip** (`7z.exe`) for all comic extractions.

### 6. The DLL-Load Failover
**Issue:** `Error 126` when loading CUDA DLLs in ONNX, causing hard crashes.
**Solution:** Wrapped GPU initialization in a `try-except` block. If DLLs are missing or incorrect, the system now automatically falls back to **CPU mode** to ensure the build never stops.

# OmnissiahCore: Developer Journey & System Documentation

## Project Vision
OmnissiahCore is a high-performance RAG (Retrieval-Augmented Generation) system specifically engineered for the complex, multi-format lore of the Warhammer 40,000 universe. The goal is to allow fans to query event chronologies, character motivations, and "narrative" lore (e.g., "What was the first alarm at the Siege of Terra?") with high fidelity across thousands of PDFs, EPUBs, AZW3s, and comics (CBR).

---

## 🛠 The Technical Stack
- **Embedding Model:** `BAAI/bge-m3` (Running via ONNX Runtime for GPU acceleration).
- **Vector Store:** `FAISS` (Index FlatIP).
- **Core Orchestration:** Python 3.10+
- **Inference:** Ollama (`qwen3:14b` / `llama3`).
- **System Dependencies:**
  - **Tesseract OCR:** For scanned PDFs and CBR comics.
  - **Poppler:** For PDF-to-Image conversion (OCR pipeline).
  - **Calibre:** For on-the-fly AZW3 to EPUB conversion.
  - **7-Zip/Unrar:** For CBR archive extraction.

---

## 📜 The "Chronicles of Error" (Developer Log)

### 1. The CUDA Compatibility Crisis (SM_120)
**Issue:** PyTorch stable lacked kernels for the RTX 5050 (sm_120), causing `AssertionError: Torch not compiled with CUDA enabled` or kernel image errors.
**Solution:** Switched to **ONNX Runtime GPU**. By exporting the model to ONNX, we bypassed the pre-compiled kernel requirement.
**Fixing DLLs:** Encountered `cublasLt64_12.dll` missing. Resolved by ensuring CUDA 12.x runtimes were in the environment path alongside CUDA 13.

### 2. The Internet Dependency Wall
**Issue:** `SentenceTransformer` and `HuggingFace Hub` would ping the internet on every load, causing the build to crash during WiFi drops.
**Solution:** Enforced strict offline mode using environment variables:
```python
os.environ["TRANSFORMERS_OFFLINE"] = "1"
os.environ["HF_DATASETS_OFFLINE"] = "1"
# And model configuration:
local_files_only=True
```

### 3. The "Denial of Wallet" Context Limit
**Issue:** Standard character-based chunking was breaking lore-heavy conversations, losing "who said what" mid-sentence.
**Solution:** Implemented **Sentence-Aware Chunking** using the NLTK `punkt` tokenizer. This ensures chunks end on actual sentence boundaries and maintain a 1-2 sentence overlap for smooth narrative flow.

### 4. The Version Mismatch Deadlock
**Issue:** `ImportError: cannot import name 'cached_download' from 'huggingface_hub'`.
**Solution:** Coordinated upgrade of `sentence-transformers` and `huggingface_hub` to resolve deprecated function calls in older library versions.

### 5. The System Path Ghost
**Issue:** Poppler and Tesseract installed on Windows but "not found" by Python even when added to system PATH.
**Solution:** Implemented **Explicit Path Injection** in `build.py`, allowing the script to look in common Windows installation directories (e.g., `C:\poppler\Library\bin`) if the system PATH fails.

### 6. The NumPy 2.0 Breaking Change
**Issue:** `AttributeError: _ARRAY_API not found` occurs because `onnxruntime` and `faiss` are not yet compatible with the newly released NumPy 2.x.
**Solution:** Downgraded the environment to use the stable 1.x branch:
```powershell
pip install "numpy<2"
```
